6-Monats-Lernplan
Webentwicklung

Vom Website-Builder zum eigenständigen Web-Entwickler

Erstellt für Sean-Lucas Chahin — April 2026

Monat 1

HTML & CSS — Die Sprache des Internets

Warum du das brauchst

Jede Website besteht aus HTML (Struktur) und CSS (Design). Deine aktuelle Website nutzt beides — aber durch Tailwind und Lovable hast du nie gelernt, wie es darunter funktioniert. Ohne dieses Fundament wirst du bei jedem Problem steckenbleiben und nicht wissen, warum etwas nicht aussieht wie gewollt.

Was du danach kannst

  • Jede Website von Grund auf bauen — ohne Tools, ohne Frameworks
  • Verstehen warum Tailwind-Klassen wie flex, rounded-2xl, py-8 funktionieren
  • Layouts bauen die auf Handy, Tablet und Desktop perfekt aussehen
  • Fehler im Design selbst finden und fixen

Woche 1–2 HTML & CSS Grundlagen

KursfreeCodeCamp — Responsive Web Design (kostenlos, interaktiv, mit Zertifikat)
Dauer~2 Stunden pro Tag
Was du lernstHTML-Tags (div, section, header, nav, a, img), CSS Farben/Abstände/Schriften, Box-Modell (margin, padding, border), CSS-Selektoren
ZertifikatJa — kostenlos nach Abschluss der 5 Projekte

Projekte die du baust

  • Tribute Page (einfache Info-Seite)
  • Survey Form (Formular mit Eingabefeldern)
  • Product Landing Page (genau wie deine WebwerkStudio-Seite)
  • Technical Documentation Page
  • Personal Portfolio

Woche 3 CSS Layouts: Flexbox & Grid

YouTubeKevin Powell — HTML & CSS for Absolute Beginners
Was du lernstFlexbox (wie flex, justify-center, items-center funktionieren), CSS Grid (für mehrspaltige Layouts wie deine Leistungen-Karten)
Warum K.P.Weltweit anerkannter CSS-Experte. Erklärt nicht nur WAS sondern WARUM

Pflicht-Übung

  • Öffne deine Index.tsx und finde jede Tailwind-Klasse
  • Schreibe daneben was sie in normalem CSS bedeutet
  • z.B.: py-8 = padding-top: 2rem; padding-bottom: 2rem;

Woche 4 Git & Terminal

KursThe Odin Project — Git Basics (3 Lektionen)
YouTubeFireship — Git Explained in 100 Seconds + 13 Advanced Git Tricks
Was du lernstCommits, Branches, Merge, git add/commit/push/pull, Änderungen rückgängig machen
Warum wichtigGenau das was wir heute gemacht haben. Ohne Git kein Deployment, keine Versionskontrolle

Pflicht-Übung

  • Erstelle ein neues Repo auf GitHub
  • Mache 10 Commits mit verschiedenen Änderungen
  • Erstelle einen Branch, ändere etwas, merge es zurück
  • Wiederhole bis git push sich natürlich anfühlt

Monat 2

JavaScript — Die Programmiersprache des Webs

Warum du das brauchst

JavaScript ist die einzige Programmiersprache die im Browser läuft. React ist JavaScript. Jeder Button-Klick, jede Animation, jede Logik — das ist JavaScript. Ohne JS-Grundlagen ist React wie Autofahren ohne zu wissen was Gas und Bremse sind.

Was du danach kannst

  • Verstehen was useState, useEffect, map(), const, => in deinem Code bedeuten
  • Eigene Logik programmieren (Formulare prüfen, Daten filtern)
  • Fehler im Code selbst finden und beheben
  • Die Grundlage für React, Node.js und alles weitere

Woche 5–6 JavaScript Basics

KursfreeCodeCamp — JavaScript Algorithms (kostenlos, ~300 Übungen)
Dauer2 Stunden pro Tag
Was du lernstVariablen (let, const), Datentypen, If/Else, Schleifen, Funktionen, Arrow-Functions (=>), Array-Methoden (map, filter, find)
ZertifikatJa — kostenlos

Konkretes Beispiel — dieser Code in deiner Website:

{services.map((s, i) => ( <motion.div key={i}> <s.icon size={20} /> <h3>{s.title}</h3> </motion.div> ))}

Benutzt: Array.map(), Arrow-Functions, Destructuring, JSX. Nach diesem Monat verstehst du jede Zeile.

Woche 7–8 JavaScript im Browser (DOM)

YouTubeWeb Dev Simplified — Learn DOM Manipulation in 18 Min
DeutschProgrammieren mit Mario — Deutsche JS-Tutorials
Was du lernstWie JS HTML-Elemente findet und verändert, Event-Listener, wie Formulare funktionieren
Warum wichtigDein Navbar-Scroll-Effekt (window.addEventListener) ist genau das

Pflicht-Projekt: To-Do-App (ohne React!)

  • Aufgaben hinzufügen, als erledigt markieren, löschen
  • Daten bleiben nach Neuladen erhalten (localStorage)
  • Nur HTML + CSS + JavaScript — kein Framework
  • Beweist: Du kannst eine Web-App komplett alleine bauen

Monat 3

React — Das Framework hinter deiner Website

Warum du das brauchst

Deine komplette Website ist in React gebaut. React ist das meistgenutzte Frontend-Framework der Welt. Wenn du React verstehst, kannst du deine Website ändern, neue Features bauen, und für Kunden komplexere Projekte umsetzen.

Was du danach kannst

  • Deine eigene Index.tsx Zeile für Zeile verstehen und ändern
  • Neue Seiten und Komponenten selbst bauen
  • useState, useEffect, Props und Components erklären können
  • React-Apps von Null starten ohne Lovable/KI

Woche 9–10 React Grundlagen

KursReact Offizielle Docs — Learn React (interaktiv, kostenlos)
Was du lernstComponents, Props, State (useState), Events (onClick), Conditional Rendering, Listen rendern (.map())
MethodeLies ein Kapitel → Mache die Übungen → Finde das Konzept in deiner Index.tsx

Tag-1-Übung: Analysiere deinen eigenen Code

  • Wie viele Components hat die Seite? (Navbar, Hero, Leistungen, Pakete, Ablauf, UeberMich, CTA, Footer)
  • Wo wird useState benutzt und warum?
  • Was macht useEffect in der Navbar?
  • Warum steht überall motion.div statt div?

Woche 11–12 React Praxis-Projekt

YouTubeWeb Dev Simplified — React Full Course
DeutschWebDev 1x1 — React-Tutorials

Pflicht-Projekt: Notiz-App mit React

  • Notizen erstellen, bearbeiten, löschen
  • Suche und Filterung
  • Kategorien / Tags
  • Daten in localStorage speichern
  • Komplett ohne KI — nur du und die React-Docs

Monat 4

Tailwind CSS & TypeScript — Deine Tools meistern

Warum du das brauchst

Du benutzt Tailwind und TypeScript bereits — aber ohne sie wirklich zu verstehen. Das ist wie mit einer Profi-Kamera fotografieren, aber nur den Automatikmodus nutzen. Wenn du diese Tools beherrschst, baust du doppelt so schnell und machst halb so viele Fehler.

Was du danach kannst

  • Tailwind-Layouts in Sekunden bauen statt in Minuten
  • TypeScript-Fehler verstehen und fixen
  • Type-safe Code schreiben (weniger Bugs)
  • Eigene Tailwind-Konfigurationen erstellen

Woche 13–14 Tailwind CSS meistern

DocsTailwind CSS Documentation — systematisch durcharbeiten
YouTubeFireship — Tailwind in 100 Seconds
Was du lernstUtility-System, Responsive Design (md:, lg:), Dark Mode, Custom Config, Animations

Pflicht-Übung

  • Baue deine WebwerkStudio Startseite komplett von Null
  • Vite + React + Tailwind — kein Kopieren, aus dem Gedächtnis
  • Ziel: unter 3 Stunden fertig

Woche 15–16 TypeScript

DocsTypeScript Handbook — The Basics
YouTubeFireship — TypeScript in 100 Seconds + Web Dev Simplified Full Course
Was du lernstWarum (i: number) in deinem Code steht, interface/type, wie TS Bugs verhindert
Warum wichtigDeine Website ist TypeScript (.tsx). Jede Editor-Fehlermeldung kommt von TS

Pflicht-Übung

  • Konvertiere deine Notiz-App aus Monat 3 zu TypeScript
  • Füge Types für jede Notiz, jeden Prop und jeden State hinzu

Monat 5

Backend, APIs & Deployment

Warum du das brauchst

Bisher baust du nur das Frontend. Aber echte Websites brauchen: Kontaktformulare die E-Mails senden, Datenbanken, Buchungssysteme. Außerdem musst du Deployment verstehen — damit du nicht bei jedem Update auf Hilfe angewiesen bist.

Was du danach kannst

  • Verstehen wie Frontend und Backend zusammenarbeiten
  • Ein Kontaktformular bauen das wirklich E-Mails verschickt
  • APIs erstellen und nutzen
  • Websites professionell deployen mit Auto-Update bei jedem Push

Woche 17–18 Node.js & Backend Basics

KursThe Odin Project — NodeJS (erste 6 Lektionen)
YouTubeTraversy Media — Node.js Crash Course
Was du lernstWas ein Server ist, HTTP-Requests (GET, POST), wie APIs funktionieren, Express.js
Warum wichtigFür das Buchungssystem in deinem Business-Paket brauchst du Backend-Wissen

Woche 19–20 Deployment & DevOps

YouTubeFireship — CI/CD in 100 Seconds
Was du lernstAutomatisches Deployment (Push = Live), Domain-Setup, SSL, Environment Variables
Warum wichtigGenau das was wir heute manuell gemacht haben — nach dieser Woche machst du es im Schlaf

Pflicht-Übung

  • GitHub Repo → Netlify Auto-Deploy einrichten
  • Eigene Domain verbinden
  • Kontaktformular mit Netlify Forms oder Formspree einbauen

Monat 6

Meisterstück — Alles zusammenbringen

Warum du das brauchst

Tutorials machen dich nicht zum Entwickler. Eigene Projekte schon. Dieser Monat ist der wichtigste — hier beweist du dir selbst, dass du es kannst.

Was du danach kannst

  • Eine komplette professionelle Website für einen Kunden bauen — alleine, ohne KI
  • Den gesamten Prozess: Planung → Code → Design → Deployment
  • Selbstbewusst sagen: "Ich bin Webentwickler"

Abschluss-Checkliste

Nach 6 Monaten solltest du jede Frage mit Ja beantworten können:

Kann ich eine Website von Null bauen ohne Lovable/KI?
Verstehe ich jeden Befehl im Terminal den ich tippe?
Kann ich meinen eigenen Index.tsx Code Zeile für Zeile erklären?
Kann ich ein Git-Repo erstellen, branchen, mergen und pushen?
Kann ich ein Tailwind-Layout in unter 30 Minuten bauen?
Kann ich TypeScript-Fehlermeldungen lesen und verstehen?
Kann ich eine Website auf Netlify deployen — ohne Hilfe?
Habe ich 3+ eigene Projekte auf GitHub die ich zeigen kann?

Die goldene Regel

Für jede Stunde Tutorial die du schaust, solltest du zwei Stunden selbst coden.
Tutorials allein machen dich nicht zum Entwickler — eigene Projekte schon.
